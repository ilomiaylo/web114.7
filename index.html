Why is the JavaScript textContent property safer than innerHTML?

The JavaScript textContent property is safer than innerHTML because we don't want unexpected HTML from a user in our site. Assigning the information to a textContent element makes it treated strictly as text.

Explain the relationship between an HTML attribute and a corresponding DOM property (explain the connection to JavaScript). Use a specific example in your answer.

Most HTML attributes can be accessed with DOM properties and methods using JavaScript. For example, if you wanted to access or change an HTML h1 element assigned the id "myHeading", you could include a script using the DOM document object:

<script> document.getElementById("myHeading") </script>

By using the document object, you or the user can make changes to the h1 element based on your site's needs.
